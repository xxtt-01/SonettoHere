# 高德地图服务领域知识

## 坐标系说明
- 高德地图使用 **GCJ-02** 坐标系（火星坐标系）
- 坐标格式为 `"经度,纬度"`（lng 在前，lat 在后），例如 `"116.481028,39.989643"`
- 不要搞反经纬度顺序，否则搜索结果会偏移到错误位置

## 技能协作流程
- **地址 → 坐标 → 路线/周边**：先用 `geocode_address` 将地址转为坐标，再将坐标传给 `nearby_search` / `get_transit_route` / `get_cycling_route`
- **模糊搜索 → 坐标 → 周边**：`fuzzy_address_search` 返回的每个 POI 含 `location` 字段，可直接作为 `nearby_search` 的中心点
- 如果用户说"明天下午出发"，先调用 `time_skill` 获取当前日期，再计算具体时间

## 搜索策略
- **具体地点**（如"北京大学"、"中关村"）→ 用 `fuzzy_address_search`
- **类别搜索**（如"附近的咖啡店"、"周边加油站"）→ 先用 `geocode_address` 获取当前位置坐标，再用 `nearby_search`
- **地址转坐标**（如"北京市海淀区中关村大街1号"）→ 用 `geocode_address`
- 模糊搜索时建议指定 `city` 参数以提高准确率，避免返回其他城市同名地点

## 路线类型
| 路线类型 | Skill | 适用场景 |
|---------|-------|---------|
| 公共交通 | `get_transit_route` | 公交、地铁换乘方案，含费用/耗时/步行距离 |
| 骑行 | `get_cycling_route` | 自行车骑行导航，含逐段转向指令 |

## 常见陷阱
- **经纬度顺序**：必须 `"lng,lat"`，反了会导致搜索中心偏移
- **城市限定**：模糊搜索不指定 city 可能返回全国同名地点，用户问"附近的肯德基"时应先获取用户位置
- **骑行 API 版本**：骑行使用 v4 接口，返回格式与公交 v3 不同（errcode vs status 字段）
- **nearby_search 的 keywords 和 types 至少必选其一**，否则 API 返回空
- **sortrule=0 按距离排序**适合"最近"场景，**sortrule=1 综合排序**适合"最好"场景
