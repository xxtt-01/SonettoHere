# SonettoHere

## Register

product

## Users

**普通用户** — 使用 AI 助手的日常用户，有一定技术背景但不一定是开发者。他们在个人电脑上使用 SonettoHere 完成信息查询、内容生成、任务规划等日常工作。使用场景多为深夜或私人时间，追求流畅、美观、不折腾的 AI 对话体验。

## Product Purpose

SonettoHere 是一个多 LLM 提供商支持的 ReAct AI Agent 桌面客户端。它让用户能够：
- 与 AI 进行自由对话，调用多种工具（天气、待办、文件操作、网页搜索等）
- 在多个 LLM 提供商之间灵活切换（DeepSeek、OpenAI、Qwen 等）
- 管理 AI 的长期记忆和个性设定
- 通过插槽技能（Skill）体系扩展 AI 能力

成功标准：用户打开就能用，配置一次就忘掉配置这件事。

## Brand Personality

**现代 · 专业 · 温暖**

- **现代** — 简洁、留白、克制。界面不喧宾夺主，让对话内容成为主角。
- **专业** — 功能完整，信息层次清晰。每个配置面板都精心组织，不暴露不必要的复杂度给普通用户。
- **温暖** — 在克制的黑白基底上，通过暖色点缀（琥珀色、暖灰）传递人味。不是冷冰冰的工具，而是有温度的助手。

## Anti-references

- **不要 SaaS 模板感** — 不采用典型 SaaS Dashboard 的大白卡 + 品牌色左边条 + 渐变背景模板。避免「企业后台」既视感。
- **不要 AI 克隆界面** — 不追随 ChatGPT/Claude 的 chat UI 范式。保持自己的视觉识别度。

## Design Principles

1. **对话即界面** — 聊天窗口是产品的核心，所有 UI 决策围绕让对话更流畅、更沉浸来展开。配置和工具是配角，不干扰主流程。
2. **克制中的温度** — 黑白基底保持专业克制，暖色点缀赋予个性。不为装饰而装饰，每个设计决策服务于可用性或情感传达。
3. **渐进式复杂度** — 面向普通用户，但保留高级功能。在界面中隐藏不必要的复杂度，让新手觉得简单，老手觉得够用。
4. **本地优先** — 桌面客户端，配置和记忆都在本地。界面应传达「私有、安全、属于你」的感觉。

## Accessibility & Inclusion

基本可用即可。确保文本清晰可读，交互元素有视觉反馈。不刻意追求 WCAG 合规，但避免明显的可用性障碍。
