# Vue 3 组件树与响应式数据流

## 什么是 Vue 组件？

Vue 组件是一个**自包含的 UI 单元**，它封装了 HTML 结构（template）、交互逻辑（script）和样式（style）。一个 `.vue` 文件就是一个组件。

本项目共有 **14 个源码文件**，其中 11 个是 `.vue` 组件，3 个是纯 TypeScript 文件。所有组件构成一棵树：

```
App.vue                        ← 根：侧边栏 + <router-view>
├── SessionSidebar.vue         ← 会话列表、创建/切换/删除
├── <router-view>              ← Vue Router 根据 URL 在此渲染页面
│   ├── ChatView.vue           ← / 路径：聊天页面
│   │   ├── StatusBadge.vue    ←   连接状态灯
│   │   ├── ChatWindow.vue     ←   消息列表容器
│   │   │   ├── MessageBubble.vue  ← 单条消息气泡
│   │   │   ├── ThinkingBlock.vue  ← LLM 思考过程
│   │   │   └── ToolCallCard.vue   ← 工具调用卡片
│   │   └── ChatInput.vue      ←   输入框
│   └── MemoryView.vue         ← /memory 路径：记忆页面
│       └── MemoryPanel.vue    ←   记忆内容展示
```

组件树是理解 Vue 应用的第一把钥匙——数据从树根流向树叶，事件从树叶冒泡回树根。

## Vue 单文件组件（SFC）结构

以最简单的 `StatusBadge.vue` 为例，认识 `.vue` 文件的三段式结构：

```vue
<!-- 1. <template> — HTML 结构 -->
<template>
  <span class="status-badge" :class="connected ? 'connected' : 'disconnected'">
    <span class="dot"></span>
    {{ connected ? '已连接' : '未连接' }}
  </span>
</template>

<!-- 2. <script setup lang="ts"> — 逻辑（TypeScript） -->
<script setup lang="ts">
defineProps<{ connected: boolean }>()
</script>

<!-- 3. <style scoped> — 样式（只影响当前组件） -->
<style scoped>
.status-badge { display: inline-flex; align-items: center; gap: 6px; }
.dot { width: 8px; height: 8px; border-radius: 50%; }
.connected .dot { background: #7ab87a; }
</style>
```

### 三段式的含义

| 块 | 语法 | 作用 |
|----|------|------|
| `<template>` | Vue 模板语法（`{{ }}`、`:class`、`v-if`、`v-for`） | 声明 HTML 结构 |
| `<script setup lang="ts">` | TypeScript + Vue Composition API | 声明状态、逻辑、props/emits |
| `<style scoped>` | 原生 CSS（`scoped` 确保样式只影响本组件） | 组件样式 |

> **`<script setup>` 是什么？** 它是 Vue 3 的"语法糖"——比传统的 `export default` 写法更简洁。在 `<script setup>` 里，`import` 的组件、定义的变量、声明的方法都自动暴露给 `<template>`，无需手动 `return`。本项目的所有组件都使用这种写法。

## Props：父组件向子组件传递数据

Props 是组件树的**唯一数据入口**。父组件通过 HTML 属性语法把数据传下去。

### 只接收一个布尔值的简单组件

`StatusBadge.vue` 只接收一个 prop：

```vue
<!-- StatusBadge.vue -->
<script setup lang="ts">
defineProps<{ connected: boolean }>()
</script>
```

父组件 `ChatView.vue` 这样传值：

```vue
<!-- ChatView.vue -->
<StatusBadge :connected="connected" />
```

`:` 前缀是 Vue 的简写，表示等号右边是 JavaScript 表达式（而非字符串）。`:connected="connected"` 等价于 `v-bind:connected="connected"`。

### 接收复杂对象的组件

`ChatWindow.vue` 接收一个 ChatTurn 数组和一个可能的 ChatTurn：

```vue
<!-- ChatWindow.vue -->
<script setup lang="ts">
import type { ChatTurn } from '@/types'

const props = defineProps<{
  turns: ChatTurn[]           // 已完成的对话轮次
  currentTurn: ChatTurn | null // 正在进行的轮次（可能为空）
  error: string | null         // 错误信息
}>()
</script>
```

模板中使用：

```vue
<template>
  <!-- 遍历已完成轮次 -->
  <template v-for="turn in turns" :key="turn.id">
    <MessageBubble role="user" :content="turn.userMessage" />
    <MessageBubble v-if="turn.finalAnswer" role="assistant" :content="turn.finalAnswer" />
  </template>

  <!-- 当前轮次（正在流式生成） -->
  <template v-if="currentTurn">
    <MessageBubble role="user" :content="currentTurn.userMessage" />
  </template>
</template>
```

关键语法：
- `v-for="turn in turns"` — 遍历数组
- `:key="turn.id"` — 每个循环项必须有一个唯一 key，帮助 Vue 高效更新 DOM
- `v-if="currentTurn"` — 条件渲染，currentTurn 为 null 时不渲染

### Props 是只读的

子组件**绝不能**修改 props。Vue 会在开发模式下报警告。如果子组件需要"修改"某个来自 prop 的值，它应该通过 emits 通知父组件去改。

## Emits：子组件通知父组件

Emits 是组件树的**事件回传通道**。子组件不直接修改父组件的数据，而是"发射事件"，让父组件决定做什么。

### 纯展示组件 + emits 示例

`ChatInput.vue` 展示了 emits 的标准用法：

```vue
<!-- ChatInput.vue -->
<script setup lang="ts">
const emit = defineEmits<{
  send: [message: string]    // send 事件携带一个 string 参数
  stop: []                    // stop 事件无参数
}>()

function handleSend() {
  const msg = text.value.trim()
  if (!msg || props.disabled) return
  emit('send', msg)           // 发射 send 事件，携带消息文本
  text.value = ''
}
</script>

<template>
  <button v-if="!isStreaming" @click="handleSend">发送</button>
  <button v-else @click="$emit('stop')">停止</button>
</template>
```

父组件 `ChatView.vue` 监听这两个事件：

```vue
<ChatInput
  :is-streaming="isStreaming"
  :disabled="!connected"
  @send="send"      <!-- 收到 send 事件 → 调用 send() -->
  @stop="cancel"     <!-- 收到 stop 事件 → 调用 cancel() -->
/>
```

`@` 是 `v-on:` 的简写。`@send="send"` 等价于 `v-on:send="send"`。

### 完整的数据流图

```
App.vue ──props(sessions, activeId)──→ SessionSidebar.vue
                                       │
                                       │ emit('create')
                                       │ emit('switch', id)
                                       │ emit('delete', id)
                                       ↓
                                     App.vue 收到事件，调用 useSession 方法

ChatView.vue ──props(turns, currentTurn, error)──→ ChatWindow.vue
              ←── 无 emits（ChatWindow 是纯展示组件）──

ChatView.vue ←──emit('send', message)──  ChatInput.vue
             ←──emit('stop')───────────  ChatInput.vue
```

## 响应式系统：ref 与 reactive

Vue 3 的核心魔法是**响应式**——当数据变化时，界面自动更新。理解 `ref` 和 `reactive` 是理解整个项目的关键。

### ref：包裹任意值的响应式容器

```typescript
import { ref } from 'vue'

const connected = ref(false)      // 包装一个布尔值
const sessionId = ref('')         // 包装一个字符串
const text = ref('')              // 包装另一个字符串

// 在 JS 中读写需要 .value
connected.value = true
console.log(connected.value)      // true

// 但在 <template> 中自动解包，无需 .value
// <span>{{ connected }}</span>   ← 直接写 connected，不用 .value
```

当 `connected.value` 从 `false` 变为 `true` 时，模板中所有用到 `connected` 的地方都会自动重新渲染。

### reactive：包裹对象的响应式容器

```typescript
import { reactive } from 'vue'

const turns = reactive<ChatTurn[]>([])

// reactive 不需要 .value
turns.push(newTurn)               // 直接操作
turns.splice(0)                   // 清空

// 模板中直接用
// <div v-for="turn in turns" :key="turn.id">
```

对比：

| 特性 | `ref` | `reactive` |
|------|-------|------------|
| 可包装 | 任意值（字符串、数字、布尔、对象） | 仅对象/数组 |
| JS 中访问 | `.value` | 直接访问 |
| 模板中 | 自动解包，无需 `.value` | 直接访问 |
| 解构保持响应式 | 否（需 `toRefs` 或手动处理） | 否（需 `toRefs`） |

### 本项目的使用规则

查看 `useChat.ts`：

```typescript
// 简单值 → ref
const ws = ref<WebSocket | null>(null)
const connected = ref(false)
const isStreaming = ref(false)

// 数组对象 → reactive
const turns = reactive<ChatTurn[]>([])
```

**模式：简单值用 `ref`，数组用 `reactive`。** 这不是唯一正解，但本项目遵循此约定。

### 响应式如何驱动 UI 更新

以一次 LLM 流式 token 为例：

```typescript
// useChat.ts 中收到 WebSocket 消息：
case 'token':
  if (turn.thinking.length > 0) {
    const last = turn.thinking[turn.thinking.length - 1]
    last.tokens += event.payload.token   // ← 修改响应式对象属性
  }
```

Vue 检测到 `turn.thinking[n].tokens` 变化 → 通知 `ThinkingBlock.vue` 重新渲染 → 用户看到新 token 出现在屏幕上。

```
WebSocket 消息 "天" 到达
    │
    ▼
last.tokens += "天"           ← 响应式数据变更
    │
    ▼
Vue 响应式系统检测到变化
    │
    ▼
ThinkingBlock.vue 重新渲染     ← <pre>{{ block.tokens }}</pre> 显示新内容
    │
    ▼
ChatWindow 自动滚动到底部      ← watch 检测到 thinking.length 变化
```

## 组件嵌套：从叶子到根的反向追踪

新手看代码时，建议从最底层的叶子组件开始追踪，逐步上升到页面组件和根组件。

### 叶子组件：MessageBubble

`MessageBubble.vue` 是整个组件树最简单的叶子——纯展示、无状态、无 emits：

```vue
<template>
  <div class="message-row" :class="role">
    <div class="bubble" :class="role">
      <div class="bubble-text">{{ content }}</div>
    </div>
  </div>
</template>

<script setup lang="ts">
defineProps<{ role: 'user' | 'assistant'; content: string }>()
</script>
```

它做的事：接收 `role`（决定是右对齐还是左对齐）和 `content`（文本），渲染为一个消息气泡。

`:class="role"` 是动态类名——当 `role` 为 `'user'` 时，渲染为 `<div class="message-row user">`，CSS 会将它右对齐并显示为米色背景。

### 中间组件：ThinkingBlock

`ThinkingBlock.vue` 接收一个 `ThinkingBlockType` 对象，展示 LLM 的思考过程：

```vue
<template>
  <div class="thinking-block" :class="{ done: block.done }">
    <div class="thinking-header">
      <span class="thinking-label">
        <span class="spinner" v-if="!block.done"></span>
        思考中{{ block.done ? '（完成）' : '……' }}
      </span>
    </div>
    <div class="thinking-body" v-if="block.tokens">
      <pre>{{ block.tokens }}</pre>
    </div>
  </div>
</template>
```

几个关键模板语法：
- `:class="{ done: block.done }"` — 当 `block.done` 为真时添加 `done` 类名
- `v-if="!block.done"` — 条件渲染：思考未完成时显示旋转动画
- `v-if="block.tokens"` — 有 token 内容时才显示文本区
- `{{ block.done ? '（完成）' : '……' }}` — 三元表达式

### 页面组件：ChatView

`ChatView.vue` 是拼装者——它不使用自己的状态，而是组合 composables 和子组件：

```vue
<script setup lang="ts">
import { useSession } from '@/composables/useSession'
import { useChat } from '@/composables/useChat'
import StatusBadge from '@/components/StatusBadge.vue'
import ChatWindow from '@/components/ChatWindow.vue'
import ChatInput from '@/components/ChatInput.vue'

const { sessionId } = useSession()
const { connected, isStreaming, turns, currentTurn, error, send, cancel } =
  useChat(sessionId)
</script>
```

`useSession()` 和 `useChat(sessionId)` 是**组合式 API（Composables）**——它们封装了可复用的状态逻辑。`ChatView` 只需调用它们并获取返回值，然后将这些值通过 props 分发给子组件。

## Vue 模板速查

本项目模板中用到的所有 Vue 语法：

| 语法 | 示例 | 含义 |
|------|------|------|
| `{{ expr }}` | `{{ content }}` | 文本插值 |
| `:prop="value"` | `:role="'user'"` | 动态属性绑定（v-bind） |
| `@event="handler"` | `@click="handleSend"` | 事件监听（v-on） |
| `v-if="cond"` | `v-if="currentTurn"` | 条件渲染（DOM 会被添加/移除） |
| `v-for="item in list"` | `v-for="turn in turns"` | 列表渲染 |
| `:class="{ class: cond }"` | `:class="{ done: block.done }"` | 条件类名 |
| `v-model="ref"` | `v-model="text"` | 双向绑定（表单元素） |
| `@keydown.enter.exact.prevent` | 见 ChatInput | 按键修饰符链 |
