# 配置迁移脚本

## 命名规则

```
from<起点版本>to<终点版本>.py
```

示例：`from0to1.py`、`from1to2.py`

## 职责

每个脚本处理本地配置从版本 N 升级到版本 N+1：
- 读取旧版本的配置文件
- 按需转换数据结构
- 写入新版本的配置文件

## 执行顺序

升级时从当前版本开始，按版本号顺序依次执行所有后续脚本。

例如当前版本为 0，目标版本为 2：
```
from0to1.py   # 0 → 1
from1to2.py   # 1 → 2
```

## 约定

- 脚本应可安全重复执行（幂等）
- 不处理跨越多个版本的升级（每个脚本只负责一个版本增量）
- 升级失败时应回滚或给出明确的错误提示
- 脚本位于 `scripts/migrations/` 目录下
- 在书写一个脚本前，应当先对local-config-manifest.yaml文件进行更新，写入新的版本号和文件修改信息
